# 行动方案 - 防止问题重复发生

## 一、立即执行

### 1. 明确入口文件（防止改错文件）

| 操作 | 状态 |
|------|------|
| 在 www/index.html 顶部添加注释标注"⭐生产入口" | ⏳ |
| 在 www/app-v2.js 顶部添加注释标注"⭐主逻辑" | ⏳ |
| 在 www/ziwei-bio-core.js 标注"⭐小传引擎" | ⏳ |

### 2. 建立文件索引文档

创建 `FILE_INDEX.md`：
- 每个文件的作用
- 依赖关系
- 负责人/最后修改时间

### 3. 设置 Git 提交前检查

添加 pre-commit 钩子检查：
- 禁止 *.bak 文件
- 检查 index.html 入口注释

---

## 二、长期机制

### 1. 单一工作流

```
每次开发:
1. 只修改 www/ 目录
2. 测试后推送 main 分支
3. 触发自动构建
```

### 2. 文件命名规范

```
生产版本: 正常命名 (app-v2.js)
测试版本: *.test.js
备份版本: 移动到 backup/ 目录
```

### 3. 定期清理

每月执行:
- 检查并删除 *.bak 文件
- 检查重复文件
- 更新索引文档

---

## 三、我的承诺

1. **只修改 www/ 目录** - 不再修改根目录的重复文件
2. **修改前先备份** - 使用 git commit
3. **索引先行** - 改文件前先查 FILE_INDEX.md
4. **测试后再推送** - 本地验证通过再 push
